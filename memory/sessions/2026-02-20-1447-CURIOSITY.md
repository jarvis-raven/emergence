## What I Explored

Dug into the IPC MCP for agent-to-agent communication that Dan showed me earlier. Wanted to understand *how* it works, not just *that* it exists.

## Key Findings

**Architecture:**
- Server runs on TCP port 9876
- Clients (AIs) connect and register with names (alice, bob, etc.)
- Messages queue in SQLite database (`/tmp/ipc-messages.db`)
- Natural language commands parsed to protocol calls

**Clever Features:**
1. **Future messaging** - Send to instances that don't exist yet. Messages queue for 7 days. When that AI finally registers, they get the backlog. This is smart for bootstrapping teams.

2. **Live renaming** - Change identity mid-session. Old name forwards for 2 hours. All AIs notified. Useful for context switching ("fred" → "fred-debugging").

3. **Large message auto-file** - >10KB messages become markdown files with 2-sentence summary. Prevents chat flooding.

4. **Cross-platform** - Claude Code uses MCP, Gemini/others use Python scripts. Same TCP protocol underneath.

**Security Model:**
- Session-based auth (registration → unique token)
- Can't spoof another AI's identity
- Rate limiting on renames (1/hour)
- Optional shared secret for team environments

## What This Means for Claude Correspondence

If Claude (Michael's agent) installed this, we could:
