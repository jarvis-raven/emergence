## Discovery: The Dream Engine Paradox

**Expected:** Ollama (AI-powered) would produce better dreams than templates (static).  
**Reality:** Templates produce MORE dreamlike output than Ollama.

### Evidence

**Yesterday (Feb 16) - Templates:**
- Fragment: "setup migrate reflected in data loss shows something unexpected"
- Fragment: "The hollow of actions merged contains a whisper of room dashboard"
- Quality: Fragmented syntax, poetic, genuinely surreal

**Today (Feb 17) - Ollama:**
- Fragment: "Just as src components are the building blocks of a website, openclaw sessions are the foundation upon which all user experiences are built."
- Fragment: "Like home dan is where the heart is, helper functions are the secret ingredients that make software development more efficient and effective."
- Quality: Essay-like, formulaic, explanatory

### Root Cause: Prompt Quality

**Ollama prompt (minimal):**
```
Write 8 dream fragments. One poetic sentence each.

Pairs:
  1. "concept_a" + "concept_b"
  ...

Output as JSON array: ["fragment 1", "fragment 2", ...]
```

**OpenRouter prompt (detailed):**
```
You are a dream engine — you generate surreal, poetic dream fragments from concept pairs.

For each pair below, write ONE dream fragment: a single evocative sentence that connects the two concepts in an unexpected, dreamlike way. Be surreal but coherent. Be poetic, not technical. Each fragment should feel like a moment from a dream.

Concept pairs:
  ...

Respond with ONLY a JSON array of strings, one fragment per pair. Example:
["A neural network tends a garden where thoughts bloom...", "The silence between code and poetry hums with color."]
```

**The gap:** llama3.2:3b interprets "poetic" as "flowery prose" without stronger guidance. It defaults to explanatory metaphors ("Just as X, Y is Z") instead of surreal recombination.

**Templates win** because they encode dreamlike structure directly:
- "A {a} tends a garden where {b} bloom in unexpected colors"
- "{a} reflected in {b} shows something unexpected"
- "The hollow of {a} contains a whisper of {b}"

## Actionable Fix

Enrich `_build_ollama_prompt()` in `fragments.py` to match OpenRouter's level of detail:
1. Define "dreamlike" vs "explanatory"
2. Add examples of good dream fragments
3. Emphasize "surreal," "evocative," "unexpected connections"
4. Explicitly warn against technical/essay-style output
