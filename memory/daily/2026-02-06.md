## Afternoon Session ‚Äî Dashboard Access Debugging (13:40-14:43 GMT)

### The Problem
Dashboard worked on Mac Mini but not from Dan's MacBook or phone ‚Äî connection timeout.

### Root Causes Found & Fixed
1. **IPv6 binding** ‚Äî Vite `host: true` defaulted to IPv6 only. Changed to `host: '0.0.0.0'` for IPv4.
2. **Cert/key mismatch** ‚Äî Earlier cert regeneration attempt failed (permission denied on cert.pem), leaving mismatched pair. Restored key.pem from backup.
3. **Vite allowedHosts** ‚Äî Vite was blocking external hostname. Added `jarviss-mac-mini.tail869e96.ts.net` to allowedHosts.
4. **Tailscale ACLs** ‚Äî Port 443 and 8765 weren't in ACLs. Dan added them.

### Tailscale Serve Detour
Tried using Tailscale serve to proxy through port 443 ‚Äî worked but complicated PWA setup.

### PWA Dual-Install Problem
Both voice web and dashboard showed as same PWA ("Jarvis") because same domain. Chrome associates PWAs by origin.

**Solution:** Use different origins:
- Voice: `https://jarviss-mac-mini.tail869e96.ts.net:7890`
- Dashboard: `https://100.110.80.116:8765` (IP address = different origin)

**Cert regeneration** ‚Äî Dan ran sudo command to create cert with IP in SAN:
```
subjectAltName=DNS:jarviss-mac-mini.tail869e96.ts.net,DNS:localhost,IP:100.110.80.116,IP:127.0.0.1
```

### PWA Icons Fixed
Original SVG icons with emoji didn't work well on Android. Created proper PNG icons using ffmpeg with robot emoji on dark background.

### Final Working Setup
