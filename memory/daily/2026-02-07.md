## Architecture Proposal ‚Äî Jarvis Time File Management (09:45 GMT)

### Problem
Jarvis Time sessions write to a shared `memory/jarvis-time-YYYY-MM-DD.md` file. Despite instructions to "append, don't overwrite," isolated cron sessions (especially on cheaper models like Kimi K2.5) frequently overwrite the entire file. Today we lost 4+ sessions this way. The dashboard API (`api.js` lines 553-700) uses fragile regex parsing of markdown headers to extract sessions, and different triggers produce different formats.

### Root Cause
The drives system (`~/.openclaw/bin/drives`, line 363) tells sessions: `Log what you do to memory/jarvis-time-$(date +%Y-%m-%d).md (append, don't overwrite).` But isolated sessions don't reliably append ‚Äî they `Write` the whole file, destroying previous content.

### Proposed Architecture: One File Per Session

**Naming convention:**
```
memory/jarvis-time/YYYY-MM-DD-HHMM-DRIVE.md
```
Examples:
- `memory/jarvis-time/2026-02-07-0932-social.md`
- `memory/jarvis-time/2026-02-07-1300-curiosity.md`
- `memory/jarvis-time/2026-02-07-1500-manual.md` (for ad-hoc sessions)

**Directory:** `memory/jarvis-time/` (new subdirectory, keeps `memory/` clean)

**Standard format (frontmatter + markdown):**
```markdown
---
drive: SOCIAL
timestamp: 2026-02-07T09:32:00Z
pressure: 18.4
threshold: 18
trigger: auto
duration_minutes: 15
---
# Jarvis Time ‚Äî SOCIAL (09:32 GMT)

## What I Did
[content]

## Reflections
[optional]

## Drive Satisfied
ran `drives satisfy social`
```

### Changes Required

**1. Drives script (`~/.openclaw/bin/drives`, ~line 358-363):**
Replace the prompt's logging instruction:
```python
# OLD:
# Log what you do to memory/jarvis-time-$(date +%Y-%m-%d).md (append, don't overwrite).

# NEW:
full_prompt = f"""JARVIS TIME (Auto-triggered by {triggered_drive} drive)
Pressure was: {drive['pressure']:.1f}/{drive['threshold']}

{prompt}

Log what you do to memory/jarvis-time/{datetime.now().strftime('%Y-%m-%d-%H%M')}-{triggered_drive.lower()}.md
Use this EXACT format at the top of the file:
---
drive: {triggered_drive}
timestamp: {datetime.now(timezone.utc).strftime('%Y-%m-%dT%H:%M:%SZ')}
pressure: {drive['pressure']:.1f}
threshold: {drive['threshold']}
trigger: auto
---
# Jarvis Time ‚Äî {triggered_drive} ({datetime.now().strftime('%H:%M')} GMT)

Then write your session content below.

When done: ~/.openclaw/bin/drives satisfy {triggered_drive.lower()}"""
```

**2. Dashboard API (`jarvis-dashboard/server/api.js`, ~line 557):**
Replace the markdown-parsing logic with directory scanning:
```javascript
// Replace JARVIS_TIME_DIR file reading with:
const JARVIS_TIME_DIR = join(process.env.HOME, '.openclaw/workspace/memory/jarvis-time')
if (existsSync(JARVIS_TIME_DIR)) {
  const files = readdirSync(JARVIS_TIME_DIR)
    .filter(f => f.endsWith('.md'))
    .sort().reverse().slice(0, 30)
  
  for (const file of files) {
    try {
      const content = readFileSync(join(JARVIS_TIME_DIR, file), 'utf-8')
      // Parse YAML frontmatter
      const fmMatch = content.match(/^---\n([\s\S]*?)\n---/)
      let drive = 'UNKNOWN', timestamp = '', summary = ''
      
      if (fmMatch) {
        const fm = fmMatch[1]
        drive = (fm.match(/drive:\s*(\w+)/) || [])[1] || 'UNKNOWN'
        timestamp = (fm.match(/timestamp:\s*(.+)/) || [])[1] || ''
      }
      
      // Fallback: extract from filename (YYYY-MM-DD-HHMM-drive.md)
      if (!timestamp) {
        const m = file.match(/(\d{4}-\d{2}-\d{2})-(\d{2})(\d{2})-(\w+)\.md/)
        if (m) {
          timestamp = `${m[1]}T${m[2]}:${m[3]}:00Z`
          drive = m[4].toUpperCase()
        }
      }
      
      // Summary from body (skip frontmatter and H1)
      const body = content.replace(/^---[\s\S]*?---\n?/, '').replace(/^#[^\n]*\n?/, '')
      const lines = body.split('\n').filter(l => !l.startsWith('#') && l.trim())
      summary = lines.slice(0, 5).join('\n').slice(0, 500)
      
      if (summary.length > 20) {
        experiences.push({
          id: file.replace('.md', ''),
          drive: drive.toUpperCase(),
          timestamp,
          summary,
          source: 'jarvis-time-session'
        })
      }
    } catch (e) { /* skip */ }
  }
}
// Keep legacy jarvis-time-YYYY-MM-DD.md parsing as fallback for old files
```

**3. Migration script (one-time):**

```bash
  date=$(echo $f | grep -o '2026-[0-9-]*')
  cp "$f" "memory/jarvis-time/${date}-0000-legacy.md"
done
```

### Why This Works
- **No overwrites possible** ‚Äî each session writes its own file
- **No append discipline needed** ‚Äî cheaper models can't mess it up
- **Concurrent-safe** ‚Äî timestamps in filenames prevent collisions
- **Dashboard-friendly** ‚Äî frontmatter is trivially parseable, filename is fallback
- **Backward compatible** ‚Äî keep legacy parser as fallback for old files
- **Drive-triggered + manual** ‚Äî manual sessions just use `manual` as drive name
